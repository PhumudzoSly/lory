# Security Policy

## Supported Versions

Security fixes are provided for the latest release on the `main` branch.

## Reporting a Vulnerability

Please do not open public issues for security vulnerabilities.

Instead, report privately to:

- Email: `phumudzomahandana+security@gmail.com`

Please include:

- Affected version or commit
- Vulnerability description
- Reproduction steps or proof of concept
- Suggested mitigation if available

We aim to acknowledge reports within 72 hours and provide regular updates
until resolution.

## Disclosure Process

- We investigate and validate the report.
- We prepare and test a fix.
- We coordinate disclosure timing with the reporter.
- We publish a release note describing the impact and remediation.
